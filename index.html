AI 赋能 自动化测试

端到端任务闭环

AI 驱动的自动化测试是利用多模态大模型实现用例自动生成、元素智能识别、用例动态优化的新一代自动化测试体系。从需求输入到反馈回流，Agent 调度各阶段任务，Skills 注入领域知识，MCP 驱动浏览器视觉定位。

多模态驱动
1

需求输入

基于测试用例文档或业务需求描述完成需求分析，推导生成明确可执行的测试意图。

2

任务编排

主 Agent 按任务类型与阶段依赖，调度分析、设计、执行等子流程。

3

分阶段执行

Subagent 各司其职：分析定规格、探索定方案、编码与验证，边界清晰。

4

领域知识注入

按阶段加载 Skills，注入 HIS 组件契约、探索规范与执行门禁，提升生成质量。

5

浏览器探索

通过 MCP 操控浏览器，自主探索页面结构与操作路径，辅以视觉定位，减少脆弱选择器依赖。

6

脚本生成

以业务意图驱动探测实现，生成抗震 Playwright 测试脚本，并通过专项回归验证后交付。

7

反馈回流

执行与回归结论自动回流上游，驱动脚本修复与持续迭代；运行中处理重试、等待、弹框等异常，对失败场景做断言校验与根因分析。

多 Agent 协助

多 Agent 协作架构

测试用例与业务需求经三阶段 Agent 串行流转：分析定规格、探索定方案、执行出脚本，阶段门禁保障质量可控。

his-test-analysis

测试分析 Agent

以 5W2H 解析用例与业务需求，识别路径包含关系并合并递进场景，产出结构化需求分析报告。

his-test-plan

测试设计 Agent

AI 自主探索页面结构与定位器，困难场景升级人机协同，产出技术设计与 Spec 测试计划。

his-test-executing

测试执行 Agent

以业务意图驱动探测实现，生成抗震 Playwright 脚本，专项验证通过后加固清理并交付。

协作机制：三阶段门禁流水线

  • 阶段一 → 二 — 需求分析报告经审查批准后，驱动浏览器探索与定位方案设计
  • 阶段二 → 三 — 技术设计与 Spec 计划经批准后，转入意图驱动的代码生成与探测
  • 探索兜底 — 同一功能自主探索连续失败 ≥3 次时，自动升级人机协同模式
  • 完成门禁 — 须通过 Playwright 专项回归验证，方可标记交付完成
