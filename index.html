多 Agent 协助

多 Agent 协作架构

Agents 区分角色边界，各司其职、协同完成从需求分析到代码生成与回归验证的全流程。

Analysis Agent

需求分析 Agent

通过测试用例、需求描述输出结构化测试报告，明确测试范围与验收标准。

Design Agent

需求设计 Agent

基于人机协作模式进行页面操作路径探索，输出设计报告和智能断言测试报告。

Coder Agent

需求执行 Agent

负责测试用例代码生成、异常修复与回归测试验证，确保脚本可稳定运行。

如何协作：Agents 区分角色边界

  • 需求分析 Agent — 解析测试用例与需求，产出需求分析报告
  • 需求设计 Agent — 人机协作探索页面路径，产出技术设计与断言方案
  • 需求执行 Agent — 生成 Playwright 测试代码，修复异常并执行回归验证
  • 三阶段门禁：每阶段产出须经审查批准后方可进入下一阶段
